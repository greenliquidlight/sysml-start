# sysml-start

> **Bootstrap SysML2 Modeling Workflow — Proof of Concept**

A text-first, docs-as-code SysML2 modeling workflow optimized for VS Code and easy onboarding of systems engineers. Model artifacts live in Git as plain text. Rendered diagrams are a day-1 capability.

---

## Purpose

This repository proves out a complete SysML2 workflow without requiring a commercial tool:

- Plain-text `.sysml` model artifacts committed to Git
- VS Code as the primary editing environment (any text editor also works)
- Rendered SVG diagrams embedded in Markdown documentation
- Command-line scripts for validate / render / build
- GitHub Actions CI for automated validation and doc generation

---

## Roles

| Role | Responsibility |
|------|----------------|
| **AI Agent** | Drafts model artifacts, views, and documentation from prompts or requirements |
| **Human (V&V)** | Reviews, validates, and approves AI-drafted artifacts via GitHub PR review |

The repository is structured so an AI agent can draft artifacts in a branch while a human performs Verification & Validation through the standard GitHub pull request process.

## Quick Start

### 1. Open in VS Code

```bash
git clone https://github.com/greenliquidlight/sysml-start.git
cd sysml-start
code .
```

When VS Code opens you will be prompted to install the **recommended extensions** (defined in `.vscode/extensions.json`). Accept the prompt.

### 2. Install Recommended Extensions

Open the Extensions panel (`Ctrl+Shift+X`) and search `@recommended` to see and install workspace-recommended extensions.

### 3. Run Tasks

Open the Command Palette (`Ctrl+Shift+P`) and choose **Tasks: Run Task**, then select:

| Task | What it does |
|------|--------------|
| **Validate Model** | Checks that all `.sysml` model files exist and would invoke a validator |
| **Render Diagrams** | Generates placeholder SVG diagrams into `generated/diagrams/` |
| **Build Docs** | Assembles an HTML index page from docs and generated diagrams |
| **Build All** | Runs Validate → Render → Build in sequence |

Or run the scripts directly from a terminal:

```bash
scripts/validate-model
scripts/render-diagrams
scripts/build-docs
```

## Rendering

Rendered diagrams are generated by `scripts/render-diagrams` and stored in `generated/diagrams/`. Placeholder SVGs are provided so documentation renders immediately, even before a real SysML2 renderer is integrated.

**To plug in a real SysML2 renderer:** replace the placeholder generation in `scripts/render-diagrams` with calls to your tool (e.g., SysML2 Pilot Implementation CLI, PlantUML, Graphviz, or similar). The `TODO` comments in the script mark the exact integration points.

---

## What is a Placeholder vs. Real

| Item | Status |
|------|--------|
| `.sysml` model files | **Placeholder** — approximate SysML2 syntax; real tooling may need adjustment |
| `scripts/validate-model` | **Placeholder** — file-existence check; wire to real validator |
| `scripts/render-diagrams` | **Placeholder** — generates simple SVGs; wire to real renderer |
| `scripts/build-docs` | **Placeholder** — assembles HTML index; wire to pandoc/mkdocs/etc. |
| Generated SVG diagrams | **Placeholder** — human-readable labels, no real layout engine |
| GitHub Actions workflows | **Placeholder** — correct structure; extend with real tool installs |
